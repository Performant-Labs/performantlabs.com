Frequently asked questions.

What is the Automated Testing Kit (ATK)?
ATK is the open-source Drupal test framework we maintain on drupal.org. It wraps Playwright and Cypress with Drupal-specific helpers — content fixtures, role-based logins, contrib-module compatibility — so a fresh test suite on a Drupal site takes hours, not weeks.
Do you work with Drupal CMS and contributed distributions?
Yes. We test stock Drupal core, Drupal CMS, and contributed distributions. If a distribution has unusual installation flow or content model, we cover it in the engagement scope.
How does a testing engagement work?
We start with a 30-minute review of your existing setup. We look at your current workflow, tell you honestly where autonomous healing would help and where it wouldn't, and leave you with a one-page writeup. No sales pitch. No obligation.
Can you test a Drupal 7 or Drupal 9 migration in flight?
Yes. Migration projects are where regression coverage matters most. We can build a parallel test suite against the legacy site, run the same suite against the new one, and surface every behavioral diff before launch.
