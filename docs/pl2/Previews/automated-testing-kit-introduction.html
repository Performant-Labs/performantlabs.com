Automated Testing Kit contains useful utilities for end-to-end testing using Cypress.io and Playwright.

If you are looking for documentation for the D7 version, go here. Looking for Testor documentation?

Join the #automated_testing_kit channel in the Slack Drupal workspace to discuss the Kit and obtain installation help.

What the Kit covers

Some of the tests and features bundled with the module:

  • logging in, logging out, and registering new users
  • password reset
  • media, node, taxonomy, and menu creation
  • search validation
  • user creation and deletion via Drush
  • cache validation
  • feed export validation
  • getting usernames and Drupal UIDs
  • awareness of the QA Accounts module
  • executing Drush commands via aliases or against Pantheon via Terminus — useful for setup and teardown

Where it runs

Automated Testing Kit works on the native OS (i.e. macOS or Linux) and on the native OS plus a container (via DDEV, Lando, or Docksal).

Note: the container we are referring to is where Drupal runs. We do not recommend attempting to run Cypress or Playwright inside a container — it makes it difficult to access the GUIs of each tool. Install Cypress and Playwright on the native OS.

Two parallel libraries

The module contains two sets of libraries and tests, one set for Cypress and one set for Playwright. Pick whichever framework your team already runs — both stacks reach feature parity for the Drupal flows the Kit covers.
