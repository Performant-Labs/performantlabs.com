What I see different (plain English)

  • Card surface (E1) — both live and preview render the four cards with a white/canvas background, fine cool-grey border, and generous inner padding. MATCH.
  • Eyebrow casing (E2) — both render "01 / Takeover", "02 / Embed", "03 / Pilot", "04 / a11y" (mixed-case, no uppercase transform). MATCH.
  • H3 trailing periods (E3) — every card title on live now ends with a period: "Test-suite takeover.", "Embedded testing engineer.", "Autonomous-healing pilot.", "Accessibility testing." MATCH.
  • Eyebrow accent metric (E5) — the terracotta accent line above each numeral cluster sits at the same vertical offset to the H3 on both live and preview. MATCH.
  • Row gap between card rows (E6) — visually equivalent on live and preview at 1280; both render the same 24-px row gap (preview's gap: var(--space-lg)). MATCH against the preview, which is canonical for layout. The issue's "~48 px" target was a measurement-with-padding artifact in the Cycle 1 audit; F's preview-aligned 24-px is correct.
  • Section H2 ("Four ways we engage.") renders smaller on live than preview at 1280. This is a typography-scale difference that is consistent across all section H2s site-wide, not a Cycle 2 target and not introduced by Cycle 2 changes. Not flagged.
  • Body copy inside each card is rendered at a slightly smaller scale on live than preview (estimated ~1 px or 6 %). Not in Cycle 2 scope; not a regression vs Cycle 1.
  • At 768 px, live renders the engagement grid as 2×2 while the preview's CSS collapses to a single column at ≤ 991 px. This is a previously-undetected responsive divergence — the Cycle 1 audit classified E4 at 768 as "MATCH" based on a different preview capture. Out of Cycle 2 scope; flagged as an advisory note for a future cycle.
  • No new WCAG regression detected at any viewport. Forced-colors, 200 % zoom, reduced-motion, and keyboard reachability all clean.

The Cycle 1 audit classified this as MATCH at 768; that was based on a different capture. Not in Cycle 2 scope. Recommend opening a follow-up cycle if operator wants preview-canonical breakpoints honored on live.
  • Section H2 + intro-paragraph typography: Live renders the H2 ("Four ways we engage.") and the intro paragraph at a smaller scale than the preview. This is consistent with the section H2 token applied across the page and not specific to engagements. The Cycle 1 audit closed the L3 typography-canon cycle as "not indicated" — that decision stands; this is a section-level typography ratio question that, if pursued, belongs in a dedicated canon cycle, not in an engagement-cards cycle.

  • Sprint 4 precedent applied: F's preview-canonical 24-px row gap was used per the source-of-truth precedence rule (brief tokens > preview layout > live; preview wins on layout) and per the AC-reinterpretation precedent set by FU-2 in Sprint 4. T validated this rationale in handoff-T. S confirms.
