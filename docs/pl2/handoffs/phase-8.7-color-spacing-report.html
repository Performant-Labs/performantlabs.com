What I see different — plain-English summary

  • Hero primary CTA pill: cyan, matches preview at all three viewports.
  • Closing-CTA primary pill (dark band): lighter cyan (dark-zone token), matches preview's dark-zone pill at all viewports.
  • Inline links and accents on light surfaces: medium teal #1893b4, matches preview's link/accent color. Feature-card title-links, FAQ accordion + glyphs, and divider accents all render the medium teal as expected.
  • Section padding: all five .padding-top--l sections render 96 px / 96 px at desktop and 64 px / 64 px at mobile, matching the preview's tighter rhythm exactly.
  • Hero kicker accents and underline rules: terracotta dashes around the kicker, single-pixel teal accents under feature-card titles — unchanged from preview, no regression.
  • Hamburger button at 375 / 768: 1 px hairline border + 8 px corner radius rendered correctly (8.6 regression PASS).
  • Header at 1280: clean 73 px row, no CTA pill, six menu links — matches the preview's lighter cluster (8.1 regression PASS).
  • Logo grid at 1280: six grayscale logos on a single row, max-height ~28 px — matches preview (8.3 regression PASS).
  • FAQ accordion plus glyph: rendered in medium teal with a 1 px right inset — matches preview (8.6 PASS).
  • Footer at 1280: three-column layout with column titles in correct casing, link copy uses warm ink color (not the new teal — by design, footer column links remain ink-toned).
About the body-height residual: live is taller than preview by +213 / +713 / +783 px at 1280 / 768 / 375. This is the operator-accepted architectural cost documented in phase 8.7 handoff (sticky-header dead-space + Dripyard component overhead + footer padding + card SDC). It is not a defect and not a verdict input for this sub-cycle. Section padding tokens themselves match the preview exactly.
About the whole-page pixel-diff numbers: they read large because live and preview have different total heights (any 1 px of vertical offset between matching elements colors that whole pixel red in the diff). Per-section visual parity is the binding check, and per-section parity is clean.
