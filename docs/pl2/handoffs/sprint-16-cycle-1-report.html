What I see different — plain-English summary

  1. REWORK Sidebar heading ("Prefer a quick call?") is much bigger on live — 32 px on live vs 22 px on the preview at desktop. The sidebar's H2 looks like it belongs to the page rather than the contained card.
  2. REWORK Form inputs on live have no autocomplete tokens. The preview has name, email, organization, tel set correctly. WCAG 2.2 SC 1.3.5 fail on live. Browser autofill won't recognise the fields.
  3. REWORK Required-field asterisk is wrong colour on live. Preview uses terracotta (#8E4A2A); live uses body text colour (~#5C544C), so the * blends in instead of standing out.
  4. REWORK Sidebar card wrapper is missing on live. Preview wraps the SavvyCal sidebar in a rounded card (hairline border, 12 px radius, 32 px padding, sticky at ≥ 992). Live renders the sidebar children loose — no card, no sticky.
  5. REWORK "What to expect" H2 is smaller on live than preview at both viewports — live = 32/24 px (desktop/mobile); preview = 40/30 px (matches brief). Visible on diff PNGs of §C at all three viewports.
  6. REWORK Closing-CTA primary button uses the wrong teal on both live and preview. Both render #62BBCB + white (held by pa11y allowlist). Brief mandates #5DC6E8 + espresso #1F1A14 (~8.81:1, passes natively).
  7. REWORK Closing-CTA buttons stack vertically on live desktop; preview shows them side-by-side. Mobile both stack — that's correct.
  8. ADVISORY Footer column headings render as H3 on live but H4 on preview. Sitewide live pattern; defensible.
  9. CARRY Hero subhead is ~2 px larger / different line-height than brief (body-lg sitewide drift — surfaced every page).
  10. CARRY Live closing-CTA primary appends a 32-px arrow icon (F-NEW-4) → 48 px pill vs preview's flat 45 px pill.
  11. PASS No orphan-word findings on this page at any viewport. Pa11y 7/7 URLs PASS.
