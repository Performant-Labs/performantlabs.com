1. What I see different

Reading from the wipe-slider comparators below — and slowing them down on hero and §E closing-CTA — these are the visible deltas an operator would notice:

  • Hero H1 size mismatch. At desktop (1280) and tablet (768) the live "Drupal testing, done by the people who wrote the tools." headline is visibly larger than the preview (live 72 px vs preview 64 px; the brief says 72). At mobile (375) the relationship inverts: preview is larger (40 px) and live is smaller (36 px); the brief says 44 px. Both renders are wrong at mobile; live is further off. F-NEW-1 / F-NEW-2
  • Closing-CTA primary button is the wrong colour on the preview. In the espresso-dark §E section, the preview shows the standard teal-on-white pill ("Book a testing review" in teal #62BBCB with white text). The live shows a different teal (#5DC6E8) with dark text (#1F1A14) — and the brief explicitly says dark-zone CTAs use the second token. Live is correct; preview is wrong. F-NEW-3
  • Primary CTA chrome differs everywhere. Every primary CTA on the live page ("Book a testing review", "Read how the workflow is wired") has a small dark-teal circle on its right edge containing a chevron arrow — a 32-px SVG suffix icon. The preview shows a plain pill with the text "Book a testing review →" — the arrow is a text glyph inside the label. Result: live pills are ~56 px tall, preview pills are ~45 px tall. F-NEW-4
  • Body copy reads at different sizes. The hero subhead and the §B / §D body paragraphs render at different point sizes on each side. Live = 20 px hero subhead, 16 px §B body. Preview = 19 / 17. Brief = 18 / 18. Both renders drift in opposite directions, so paragraphs wrap at different line counts and everything below cascades vertically. Sprint 13 carry-forward
  • Open-source cards have different call-to-action structure. The live cards make the title a link ("Automated Testing Kit (ATK)" is clickable). The preview cards have a separate "Read the docs →" link at the bottom. Both communicate the same intent; the preview's pattern is more visible but the live's is what ships. Sprint 12 carry-forward
  • Footer is much richer on the live than the preview. The Drupal-shipped footer has services / resources / company columns plus a privacy strip and "Get in touch" — the preview footer is single-row. This is sitewide and out of /about-us scope. Sitewide
  • Header looks identical now — the Sprint 13 preview canonicalization landed; the right-side "Book a testing review" pill is gone from the preview and the hamburger toggle is shown below 992 px. No header delta this cycle.

6. Findings catalog

F-NEW-1 — Preview hero H1 desktop short. Pre-committed (runbook)

Brief display-xl = 72 px / line-height 1.05 / letter-spacing −2 px desktop. Live renders 72 / 79.2 / −1.8 ✅. Preview renders 64 / 67.2 / −1.6 ❌.

Layer: Preview-doc. Fix: raise .hero h1 { font-size } on line 254 of docs/pl2/Previews/about-us.html from 64 to 72 px; adjust letter-spacing to −2 px.

F-NEW-2 — Live hero H1 mobile short. Pre-committed (runbook)

Brief mobile display-xl = 44 px at < 576 px. Live = 36 ❌. Preview = 40 (also short).

Layer: L1 token + preview-doc. Cross-page sweep required (PC-3): any landing page with a display-xl hero will gain 8 px live / 4 px preview vertical headline height at mobile.

F-NEW-3 — Preview §E closing-CTA primary token wrong. NEW this cycle

Brief line 319 explicitly defines dark-zone primary CTA: #5DC6E8 bg + #1F1A14 text (8.81:1 AA pass). Live applies this correctly ✅. Preview applies the light-zone token #62BBCB + white ❌.

Layer: Preview-doc. Fix: update docs/pl2/Previews/about-us.html .section--espresso .btn--primary (or equivalent selector) to background: #5DC6E8; color: #1F1A14;.

F-NEW-4 — Primary CTA component chrome differs. NEW — operator decision needed

Live uses dripyard_base:button component → appends 32-px SVG chevron-circle suffix via .button__suffix → 56-px-tall pill. Preview uses plain <a class="btn--primary"> with inline "→" glyph → 45-px-tall pill.

Brief does not enumerate the suffix-icon. Three options:

  1. Option A — preview canonical. Strip suffix-icon from live (component / template work — high cost, sitewide impact).
  2. Option B — live canonical. Update preview to render chevron-circle suffix (preview-doc only).
  3. Option C — accept structural difference and document in brief as "Drupal-component-templated CTA suffix-icon is shipped behavior, not part of visual-fidelity contract" (recommended for autonomous park).
F-NEW-5 — Live display-md H2 line-height +1.2 px vs brief. Minor

Brief 40 / 1.05–1.10. Live computed 40 / 45.2 (1.13). Preview 40 / 44 (1.10) ✅. Defer to Sprint-15+ typography pass; sub-perceptual at this delta size.

Hero subhead: live 20 / preview 19 / brief 18. §B / §D body: live 16 / preview 17 / brief 18. Sitewide token surface — recommend a separate body-lg sitewide cycle, not in Sprint 14 scope.
