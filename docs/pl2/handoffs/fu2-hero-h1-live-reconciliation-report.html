What I see different

For the audit-relevant element (the hero H1 typography), there are no visible differences between live and preview at any viewport. Multi-line wrap behavior, character size, letter-spacing, line-height, weight, and color all match across all four pages and both viewports.

The diff PNGs do show large red regions, but they trace to pre-existing, out-of-scope differences:

  • Hero band background color. Live: cream/off-white. Preview: pure white. Affects the whole hero band on the three non-home pages. Pre-existing — not introduced by FU-2.
  • Breadcrumb row. Live: "Home > Services" (or similar) above the hero. Preview: no breadcrumb. Pre-existing CMS chrome.
  • Header right-side CTA pill. Preview: "Call today" / "Book a testing review" pill. Live: no pill (canonical header per the design memory). Pre-existing from prior header reassessment cycle.

None of those are typography concerns and none were touched by FU-2. The Cycle 3 divergence (-1.8px / 1.10 on the three non-home pages) is gone — live now matches the brief and the previews on H1 metrics.
