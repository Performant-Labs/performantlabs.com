What I see different

In scope for this sub-cycle (8.5)

  • PASS Hero band on live no longer carries the ~600 px excess whitespace below the CTAs. At 1280 and 768 the gap from the bottom CTA to the "TRUSTED BY TEAMS AT" label is 96 px on both live and preview.
  • PASS At 375, the gap is 64 px on both live and preview, matching the mobile var(--space-3xl) token in the preview.
  • PASS The dy-section wrapping the logo grid now starts immediately below the hero — there is no longer a tall white gap between hero bottom and logo-grid label.

Out of scope (other sections that still differ — sub-cycles 8.1, 8.3, 8.6)

  • OUT-OF-SCOPE 8.1 Header on live is taller (~160 px) than preview (~73 px) and uses a different nav layout (separate nav links vs. compact "Call today" button on preview). The whole hero+logo-band region is shifted down on live by roughly 87 px relative to preview because of this; the spacing within the region itself is correct.
  • OUT-OF-SCOPE 8.6 Logo presentation differs: live shows real logo images (CBS, DocuSign, Orange, Renesas, Robert Half, Tesla) in a 1- or 2-column grid at narrower widths; preview shows logos as gray text labels in a 6-up row at 1280 and a 2-column grid at 375. This is a logo-grid presentation choice, not a spacing concern.
  • OUT-OF-SCOPE 8.3 FAQ accordion icons, footer-CTA polish, and footer casing further down the page also still differ — these contribute to the whole-page delta (43–51%) but not to the hero+logo-band crop in scope here.
  • OUT-OF-SCOPE Hero background image: live carries an actual background-image; preview uses a flat white. This contributes pixel delta inside the hero crop but does not affect spacing.
