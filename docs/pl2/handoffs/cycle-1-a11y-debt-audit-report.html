Thread 2 — FU-7b root cause

Why the skip only manifests on /articles

Emitting file

web/themes/custom/performant_labs_20260502/components/article-card/article-card.twig line 37:

<h3><a href="{{ url }}">{{ title }}</a></h3>

The SDC itself is not the bug. The SDC's H3 is correct in 5 other consuming pages because each wraps the card in a section H2. The skip only manifests where the Views render path inserts cards directly under H1 — /articles.

Recommended fix (deviates from runbook default)

Edit web/themes/custom/performant_labs_20260502/templates/views/views-view-unformatted--articles--page-1.html.twig to insert a visually-hidden <h2> above .article-grid:

{% if title %}
  <h2>{{ title }}</h2>
{% else %}
  <h2 class="visually-hidden">Articles</h2>
{% endif %}
<div class="article-grid">
  {% for row in rows %}
    {{- row.content -}}
  {% endfor %}
</div>

Why not "promote h3 to h2" (the runbook default): the SDC is shared by 5 unrelated pages. Promoting in-SDC cascades hierarchy changes to all of them. Per-page conditionals inside the SDC add prop sprawl. The Views template override is the per-page scoping mechanism Drupal provides for exactly this case.
