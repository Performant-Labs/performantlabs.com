Process gap surfaced for Sprint 16 wrap

The Cycle 1 audit probe script measured the wrong selectors and produced two false-positive findings (E + F).

  • scripts/sprint-16-cycle-1-probe.mjs collected only h2s[0] (sidebar "Prefer a quick call?") and h2s[h2s.length-1] (§D closing-CTA H2). The §C "What to expect" H2 (3rd content H2 in DOM) was never directly measured. Its size was inferred from a DSSIM-based section diff, which actually reflected layout / chrome differences, not an H2 type-scale problem.
  • For §D, the Cycle 1 probe measured the form submit button (rgb(98,187,203) + white) and attributed those values to the closing-CTA primary as well. The closing-CTA primary a.button--primary inside .dy-section--cta-pair was already on the theme--dark .button--primary cascade and already resolved to #5DC6E8 + #1F1A14.

Recommended remediation for future audits. Audit probe scripts must:

  1. Enumerate all <h2> elements with their text and section ancestor — not just [0] and [-1] by array index.
  2. Probe each section's primary CTA by section ancestor (e.g. .dy-section--cta-pair .button--primary), not the first .button--primary found in the page.
  3. When a per-section DSSIM signals "real visible delta" but the per-token probe doesn't surface a value mismatch, flag for follow-up rather than reverse-inferring the cause — DSSIM-on-section is a sensitive but not specific signal.

Three of the seven Cycle 1 findings (D-marker, E, F) turned out to be probe artefacts. The remaining four (A, B, C, G) were real and have been merged in Cycles 1–3.
