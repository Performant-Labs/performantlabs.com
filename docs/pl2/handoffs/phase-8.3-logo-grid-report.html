What I see different (plain English)

  • Logo bar wrap pattern matches preview at every viewport. 1280: single row of 6. 768: 4 logos top, 2 bottom (the brief's "two rows of three" is stale). 375: 2+2+2. This resolves F's open question.
  • Logo treatment matches preview exactly. Each logo on live has the same grayscale level, the same 0.7 opacity, the same 28 px height, and the same set of six logos in the same order as the preview. Visually the bar reads identically.
  • Horizontal positions of individual logos differ between live and preview. On live each logo is centered within a uniform 140 px cell; on preview each logo uses its intrinsic width (CBS 145 px, Robert Half 200 px, Tesla 22 px). This shifts each logo by a few dozen pixels horizontally. This is an accepted infrastructure constraint documented in F's handoff: Drupal's Canvas responsive-image template requires an explicit CSS width or images render at 0×0.
  • Vertical gap above the logo bar differs from preview. On live at 768 there is ~140 px of whitespace between the "TRUSTED BY TEAMS AT" label and the first logo row; on preview the gap is ~20 px. This is hero/section padding, not a logo-bar concern, and is out of scope for 8.3. Flag for 8.6 footer/polish cycle if desired.
  • Whole-page screenshots are ~1000 px taller on live than preview at every viewport. This is not a logo-bar issue. It reflects section spacing differences in hero, feature cards block, and footer/CTA bands that are owned by 8.6 (next cycle).
  • No regressions detected on prior fixes (8.1 header pill, 8.2 hero padding-inline, 8.4 feature card 1-col stack at 768, 8.5 hero min-height).
