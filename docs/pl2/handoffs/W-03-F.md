# Handoff-F: Phase W-03 - Tokenise hardcoded -150px margin in header-article.css

**Date:** 2025-06-09
**Branch:** `aa/pl-neonbyte-W-03-header-article-margin`
**Issue:** W-03

## What was done
- Modified `themes/neonbyte/components/header-article/header-article.css` — declared `--header-article-image-offset: 150px` as a component-scoped CSS custom property on `.header-article__image[class]` and replaced the hardcoded `margin-top: -150px` with `margin-top: calc(-1 * var(--header-article-image-offset))`.

## Layer decisions
**Layer 5 (component stylesheet)** — The variable is scoped to `.header-article__image[class]`, a component-level selector. This is the highest correct layer because:
- The offset is specific to the article header image component, not a theme-wide token.
- The constraint explicitly forbids introducing a new token at the theme layer.
- Layer 3 (theme layer) would be too broad; Layer 1 (config) is not applicable to CSS custom properties.

## Architecture notes for A
- Single-file change. No new dependencies introduced.
- The component-scoped variable `--header-article-image-offset` defaults to `150px`, preserving the existing rendered offset.
- Parent components or themes can override the offset by targeting `.header-article__image[class]` and setting a different value for `--header-article-image-offset`.
- No Canvas component_version handling required — this is a pure CSS change.

## Deviations from spec
None. The implementation matches the brief exactly:
- A CSS custom property is declared in `.header-article__image[class]` defaulting to `150px`.
- `margin-top: -150px` is replaced with `margin-top: calc(-1 * var(--header-article-image-offset))`.
- No existing spacing token in `variables-layout.css` equals `150px` (closest is `--spacing-xxxl` at 120px), so a new component-scoped variable was introduced as specified.
- No other files changed.
