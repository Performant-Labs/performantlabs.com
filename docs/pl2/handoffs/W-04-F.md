## Architecture notes for A

- The z-index ladder in the neonbyte theme is: hero content (1) < dropdown menus (5) < header (10) < mobile-nav-button (11) < `.region--fixed-middle-right` (20). The gap between 11 and 20 is intentional headroom. A new fixed element added in this range would need to be slotted in consciously.
- `--offset-from-header` is written via `setAttribute('style', ...)` directly on the `header-navigation-wrapper` element, meaning it is scoped to that element and cascades inward. It is not set on `<html>` or `<body>`. If the header layout gains new right-side items (CTA pill, language switcher expansion), the JS calculation automatically adapts; the CSS fallback of 250px does not.
- `--space-for-fixed-header` is a global token consumed in two places within `.hero--position-against-screen-top`: once on the hero element itself (padding) and once via a `:has()` selector reaching the parent section (negative margin). These are cross-element in the cascade; a future refactor separating the hero from its layout parent would need to replicate or re-home the negative margin rule.
