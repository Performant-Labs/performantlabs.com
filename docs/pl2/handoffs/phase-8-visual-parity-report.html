What I see different

Operator-readable list of every visible delta. Status pills indicate severity.

  • REWORK Header. Live (1280) is much taller and shows a full primary nav (Services, How we do it, Articles, Open source projects, About us, Contact us) plus a large "Book a testing review" pill. Preview shows only the wordmark and a compact "Call today" pill — no nav.
  • REWORK Header (mobile). Live shows hamburger only, no CTA. Preview shows wordmark + "Call today" pill, no hamburger. The two implementations have opposite responsive patterns.
  • REWORK Hero headline size (desktop). Live renders "Ship Drupal releases with confidence." at roughly 120 px (display-2xl-equivalent). Brief specifies display-xl ~72 px, which is what the preview uses.
  • REWORK Hero headline overflow (tablet, 768). Live's headline runs past the right viewport edge — the screenshot shows the word "releases" cut off mid-letter. Preview wraps cleanly within the container. This is a missing max-width or font-size cap on the heading at the tablet breakpoint.
  • REWORK Hero whitespace. Live adds ~600 px of empty whitespace below the hero CTAs at desktop, before the logo band. Preview has the logo band directly below.
  • REWORK Feature cards (desktop, 1280). Live renders cards in a 2 + 1 layout (Tools and AI on row 1, People alone on row 2). Brief and preview both use a single 3-column row at desktop.
  • REWORK Logo grid (mobile, 375). Live stacks bitmap logos vertically (one logo per row) into a very tall column. Preview uses a 3 × 2 grid of plain text names. The two implementations use entirely different responsive strategies — operator decision needed.
  • REWORK Logo grid (tablet, 768). Live's bitmap logos overflow past the right viewport edge; only ~5 of 6 logos are visible, last one clipped. Preview shows the same six as text names in a single fitted row.
  • DELTA Footer-CTA primary pill. Live's "Book a testing review" pill includes an inline arrow glyph (→). Preview's pill has no icon. Decide canonical treatment.
  • DELTA FAQ accordion icons. Live uses chevron-down (∨), preview uses plus (+). Pick one.
  • DELTA Footer link label casing. Live: "Test Suite Takeover", "Embedded Testing Engineer". Preview: "Testing-suite takeover", "Embedded testing engineer". Inconsistent casing scheme.
  • DELTA Built-for-team checklist items. Preview ends each item with a period; live drops the period. Minor copy inconsistency.
  • MATCH Heal-flow ("We heal our own tests nightly."). The four-step pill row, terracotta step numbers, teal arrows, and the highlighted step-04 outline all render the same on both. Outer card chrome and dogfooding kicker also match.
  • MATCH Card kicker treatment. Where the cards do render, the terracotta short-rule plus "01 / TOOLS" eyebrow plus corner arrow icon match between live and preview. Phase 3 (kicker SDC) and Phase 4.1 (card override) are visually intact.
  • MATCH Hero CTA pair styling. Filled teal pill + outline teal pill with teal text — matches between live and preview at desktop.
  • MATCH Footer-CTA dark band & "BOOK A REVIEW" kicker. Both render correctly with the espresso surface, terracotta kicker rules, and large headline.

Methodology & reproduction

  1. Verified default theme: ddev drush cget system.theme default → 'system.theme:default': performant_labs_20260502.
  2. Started preview server: cd docs/pl2/Previews && python3 -m http.server 8765 &.
  3. Captured six full-page PNGs via Playwright (chromium.launch() → newContext({viewport}) → page.goto({waitUntil:'networkidle'}) → 800 ms settle → page.screenshot({fullPage:true})) at 1280×800, 768×1024, 375×667 against the live URL and the local preview URL. Saved under docs/pl2/handoffs/screenshots/cycle-8/.
  4. Padded each pair to a common canvas with magick … -background white -gravity northwest -extent {W}x{H} (because the live and preview heights differed).
  5. Ran compare -metric AE to compute pixel differences and emit the diff PNG; computed delta % as (pixels / (W*H)) * 100.
  6. Built side-by-side composites with magick +append.
  7. Cropped section thumbnails (magick … -crop WxH+X+Y) and embedded them in this report.
  8. Stopped preview server.

Full handoff with verdict, rework list, and design-brief compliance table: phase-8-visual-parity-S.md.
